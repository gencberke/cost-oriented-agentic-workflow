# Review package + report contract

Loaded on demand. The reviewer receives **bounded artifact paths**, never large
raw logs in the dispatch prompt. Two zero-dependency helpers own the evidence:

```text
skills/requesting-review/scripts/review-package.mjs   build | validate | inspect
skills/requesting-review/scripts/review-report.mjs    validate | inspect | render | summarize-findings
```

The diff itself is generated by the bash `execution-routing/scripts/review-package`
(committed BASE..HEAD, or task-scoped with `-- PATH...`); the `.mjs` builder writes
the JSON **descriptor** that points at that diff plus the brief, baseline, report,
and verification summary.

## Dispatch fields (named on every `cow-reviewer` dispatch)

Required: `REVIEW_SCOPE`, `MODE`, `RISK`, `REVIEW_PACKAGE_PATH`,
`REVIEW_REPORT_PATH`, `WORKTREE_ROOT`, `REVIEW_TARGET_ID`. When applicable:
`PLAN_PATH`, `LEDGER_PATH`, `UNIT_BASELINE_PATH`, `UNIT_REPORT_PATH`,
`UNIT_DIFF_PATH`, `VERIFICATION_SUMMARY_PATH`, `PRIOR_REVIEW_REPORT_PATH`,
`ACCEPTED_FINDING_IDS`, `REMEDIATION_WAVE`, `BASE_SHA`, `HEAD_SHA`.

## Package descriptor (schema v1; ≤ 8 KB; references artifacts, never inlines bulk)

A review package serves as bounded, controller-provided input evidence for the reviewer.

- **Unit:** `reviewScope, reviewTargetId, mode, risk, taskBriefPath, planPath,
  unitBaselinePath, implementationReportPath, acceptedAttempt, unitOwnedPaths,
  allowedPaths, knownPreExistingPaths, baseSha, headSha, diffArtifactPath,
  verificationSummaryPath`.
- **Targeted re-review:** `priorReviewReportPath, acceptedFindingIds,
  remediationWave, changedPathsSinceReview, verificationSummaryPath`.
- **Whole-work:** `planPath, ledgerPath, unitCommitList, baseSha, headSha,
  diffArtifactPath, verificationSummaryPath, unitReviewReportPaths,
  findingLedgerPath`.

Build with `review-package.mjs build --scope … --target … --mode … --risk …
--output <pkg> [path/sha flags]`; it validates per-scope required fields and
rejects unsafe paths before writing.

## Report (schema v1; ≤ 12 KB)

The reviewer is read-only (no Write tool): it returns **one fenced JSON object**.
The report serves as reviewer-produced output evidence, including `remainingRisks[]`.
The controller persists it to `REVIEW_REPORT_PATH` and runs
`review-report.mjs validate <report> --package <pkg> [--accepted-finding-ids …]`
**before** adjudicating. Run `review-report.mjs validate` before adjudication or remediation. Fields:

```text
schemaVersion(1) reviewScope reviewTargetId mode risk
specVerdict(PASS|CONCERNS|FAIL) qualityVerdict(PASS|CONCERNS|FAIL)
overallVerdict(APPROVE|CHANGES_REQUIRED|BLOCKED)
findings[]{ id severity(CRITICAL|IMPORTANT|MINOR)
            causality(INTRODUCED|WORSENED|PRE_EXISTING|UNCERTAIN)
            status(OPEN|RESOLVED|NOT_RESOLVED|OUT_OF_SCOPE)
            path line title evidence recommendation blocking }
reviewedArtifacts[] remainingRisks[]
```

Validation rejects: malformed JSON, unknown schema version, unsafe paths,
duplicate finding ids, invalid severity/causality/status, a **blocking Minor**, a
**blocking PRE_EXISTING/UNCERTAIN** finding (only INTRODUCED/WORSENED may block),
`APPROVE` with an open blocking finding, an in-unit finding outside package scope
without `OUT_OF_SCOPE`, a targeted re-review that omits an accepted finding or
carries a non-accepted non-introduced finding, oversized reports, and smuggled
code fences / diffs / secrets / chain-of-thought. A failed report is preserved as
evidence; the helper never edits source.
